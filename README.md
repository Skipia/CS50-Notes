## Introdução
Este repositório nasceu como um **hobby** e um desafio pessoal. Aqui, registro minha jornada através do **CS50x: Introduction to Computer Science** da Universidade de Harvard. 

Atualmente estou realizando os estudos do CS50x na versão de 2024.
- Então, após finalizar o curso, pretendo estudar as versões mais recentes também, para verificar se há algo a ser acrescentado nas anotações. 
- Com isso, para evitar arquivos e textos repetidos em um determinado tema, irei apenas incrementar novas informações às notas e atualiza-las.

O objetivo deste espaço é centralizar meus estudos sobre lógica de programação, algoritmos e estruturas de dados, enquanto aprendo na prática a dominar ferramentas como **Git**, **GitHub** e o fluxo de trabalho com o **Obsidian**.

## Habilidades em Desenvolvimento
Este projeto foca na transição do pensamento comum para o pensamento computacional. Algumas habilidades trabalhadas:
* **Lógica de Programação:** Entendimento de funções, loops e condicionais.
* **Gerenciamento de Memória:** Manipulação de ponteiros e alocação dinâmica em C.
* **Estruturas de Dados:** Arrays, Listas Ligadas, Árvores e Tabelas Hash.
* **Versionamento de Código:** Git (Commits, Push, Branching e .gitignore).

---

## Estrutura do Repositório
As notas estão organizadas por semanas, seguindo o cronograma do curso:
* `Week 0 - Scratch`: Lógica visual e conceitos iniciais.
* `Week 1 - C`: Sintaxe básica e tipos de dados.
* `Week 2 - Arrays`: Organização de dados em sequência. (Não Finalizado)
* *(Em progresso...)*

**OBS: L1 = Lesson 1**
- Há módulos originalmente sem numeração. Então, para manter a ordem do cronograma, eu coloquei como "sub-módulos" (Ex: 6,5). Contudo, isso não indica necessariamente uma interligação direta entre os conteúdos.

## Contribuindo

Como este é um repositório de estudo, **correções são sempre muito bem-vindas!** Se você notar algum conceito que expliquei errado ou um erro de digitação nas notas:

1. Faça um **Fork** do projeto.
    
2. Crie uma branch para sua modificação (`git checkout -b feature/correcao-conceito`).
    
3. Abra um **Pull Request** explicando o erro e a correção.
    

- Também aceito dicas de boas práticas de código.
